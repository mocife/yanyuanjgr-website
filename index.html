Project Background

01 项目背景

城市化进程过快，规划决策与实际需求脱节，存量公共空间存在诸多问题。城市规划复杂且门槛高，公众参与深度不足。

"规划不是自上而下的蓝图，而是自下而上的对话。"

🎯

打破空间壁垒

虚拟城市空间已成为现实空间的镜像与延伸，两者通过"感知、再组织、再认识、反馈"的环状路径产生深度互动。

🚪

降低参与门槛

严肃游戏融合互动机制与情境模拟，采用交互叙事方法，让全年龄段都能参与复杂规划，实现城市治理权利的下放。

⚡

提升参与效率

游戏化体验增强归属感与认同感，虚拟界面让居民直接进行空间布局，数据收集效率远高于传统调研。

Game Design

02 游戏设计

🎮 《燕园主理人》— 经营类放置装修游戏

玩家以北大学生的身份，在校园中探索、发现各类空间问题，通过资源管理和创意设计，亲手改造并激活校园空间，提升师生满意度和校园活力。
🗺️北京大学景观学院与城环学院园区
⏱️15-30分钟完成"发现-改造-效益"循环
📱安卓端 + Windows 端
📍

物理空间定位与诊断

采用地理围栏算法构建经纬度矩形包围盒，设置0.0002°容差范围过滤GPS漂移。"你画我猜"式交互验证机制，激励玩家仔细观察场地细节。

🏗️

网格建造系统

130+物品库，4大类（公共设施/装饰艺术/景观植被/道路铺装），支持放置、旋转、删除、叠加操作。智能管理模块实时计算物品在网格上的精确坐标。

🤖

AI NPC 智能对话

DeepSeek API 驱动的智能体对话 NPC，角色Raby活泼可爱。Temperature采样控制输出随机性，多轮对话支持，任务限定装扮系统。

☁️

云端数据存储

LeanCloud BaaS 支持，建造数据云端同步，好友校园互联互通，本地数据持久化存档机制。

Competitive Analysis

03 竞品分析

🗺️

探索导引类

以 LBS 地理位置服务为核心，通过 GPS 寻找虚拟/实体宝藏。

代表：Pokemon GO、Geocaching

优点：提升空间"到访率"与"在场性"

缺点：缺乏对空间实体评价，难建立场地认同

LBS + GPS
🏗️

模拟建造类

提供上帝视角与高度自由的编辑工具，在虚拟世界建设城市。

代表：我的世界、模拟城市

优点：赋予玩家极高的设计权与创造空间

缺点：难以直接转化为现实

沙盒建造
🎭

叙事体验类

结合实物解谜与实地线索，将空间转化为文化剧场。

代表：各类剧本杀

优点：推动文化传播，增强玩家情感连接

缺点：互动往往是线性的、一次性的打卡

沉浸式叙事
🌱

行为回馈类

"行为积累→数值转化→实体回馈"逻辑，激励持续参与。

代表：TrainPal Trees、蚂蚁森林

优点：有效解决公众参与动机不足问题

缺点：界面扁平化，缺乏对物理空间三维感知

激励机制

现有城市模拟经营类游戏，与公众参与存在路径壁垒，多限于虚拟，缺少现实价值。
能否开发一款游戏，作为公众参与社区治理的新媒介？
"体验化、游戏化、低门槛、高互动"

道路铺装石板路、碎石路使用频率较低

🧠 空间聚类结果

🌿 Cluster 0：景观导向型

以绿色植被为主导，景观丰富的空间类型，绿地占比例高。

🏢 Cluster 1：集中核心型

建筑密度较高、结构相对集中的空间布局，建筑比例占优。

🛤️ Cluster 2：交通导向型

道路结构复杂、空间较为碎片化的区域，道路边缘复杂度高。

🔬 研究方法

图像预处理

统一尺寸保留核心画面，RGB→HSV颜色空间转换，阈值分割提取绿地/道路/建筑。

特征量化

绿地占比、道路占比、建筑占比，道路边缘复杂度等空间特征指标构建。

K-means 聚类

特征标准化后无监督聚类，将校园空间分为三种主要类型进行分析。

Future Vision

06 未来愿景

连接现实

以游戏为桥，让"空间共创"连接更真实、走进更多场景。

🏘️

拓展边界

从校园建设推广至老旧小区改造等更广泛的城市微更新领域。

💡

输出方案

提供一套低门槛、激发创意的"公众参与"设计解决方案。

📦

👥

💰

🐛

稳定性迭代：系统中仍存在一些bug，正在持续修复中。

📲

iOS 端：由于开发者证书签名与打包流水线的复杂性，目前尚未成功生成测试包。
